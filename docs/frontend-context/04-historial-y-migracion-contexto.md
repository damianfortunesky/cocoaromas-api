# Frontend — Historial y migración de contexto

## Objetivo de esta consolidación
Reducir la fragmentación de contexto en múltiples `task-*.md` y concentrar la información funcional/técnica en documentos orientados a operación.

## Qué se consolidó
- Arquitectura base y estructura modular.
- Contratos API realmente consumidos por frontend.
- Reglas de negocio que impactan UX/flujo.
- Adaptaciones de frontend ante cambios del modelo de datos backend.

## Resultado
- Menos archivos para onboarding.
- Menor ambigüedad entre pedidos históricos repetidos.
- Contexto más directo para mantenimiento y próximas iteraciones.
